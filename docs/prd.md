# PRD - HoloID

## 1. 프로젝트 개요

"나를 증명하는 단 한 장의 시네마틱 홀로그램" — 손에 쥐고 있는 듯한 입체감과 화려한 빛 반사를 제공하는 인터랙티브 3D 디지털 명함(ID 카드) 제작 및 공유 플랫폼이다.

카카오톡 오픈채팅방과 같은 익명 커뮤니티에서 자신을 재미있고 강렬하게 소개하기 위한 바이럴 콘텐츠로 활용된다. 사용자의 선택이 화면에 100% 실시간으로, 화려하게 반영되는 것이 핵심 경험이다.

## 2. 목표 사용자

- 카카오톡 오픈채팅방, 디스코드 등 익명 커뮤니티에서 자신을 개성 있게 소개하고 싶은 사용자
- 개발자, 디자이너 등 자신의 정체성을 재미있게 표현하고 싶은 크리에이터
- 바이럴 콘텐츠에 관심 있는 MZ세대 소셜 미디어 사용자

## 3. 핵심 기능

- [ ] 기능 1: **인터랙티브 카드 뷰어** — 자이로/마우스 반응형 틸트, 광택/무지개빛 반사, 카드 뒤집기, 패럴랙스 뎁스 효과
- [ ] 기능 2: **카드 에디터 (Creator Studio)** — 카드 재질(포일, 네온, 글래스, 메탈) 선택, 캐릭터 아바타 커스터마이징, 스탯 레이더 차트, 칭호 뱃지, 파티클/오라 이펙트 선택

### 기능 2 상세: 캐릭터 아바타 커스터마이징 시스템

도트(픽셀) 아트 스타일의 캐릭터 아바타를 14개 카테고리에서 자유롭게 조합한다.

**아바타 파츠 카테고리 (14개):**

| 그룹 | 카테고리 | 옵션 수 | 예시 |
|------|---------|---------|------|
| 얼굴 기본 | 피부톤 | 8 | 밝은살구, 밀색, 갈색 등 |
| 얼굴 기본 | 얼굴형 | 6 | 둥근형, 각진형, 계란형 등 |
| 얼굴 기본 | 눈 | 12 | 기본, 고양이눈, 하트눈 등 |
| 얼굴 기본 | 눈썹 | 8 | 일자, 아치형, 굵은 등 |
| 얼굴 기본 | 코 | 6 | 기본, 작은코, 높은코 등 |
| 얼굴 기본 | 입 | 10 | 미소, 무표정, 활짝웃음 등 |
| 헤어 | 헤어스타일 | 20 | 숏컷, 포니테일, 웨이브 등 |
| 헤어 | 헤어컬러 | 12 | 검정, 금발, 핑크, 은색 등 |
| 악세사리 | 안경 | 8 | 없음, 둥근안경, 선글라스 등 |
| 악세사리 | 귀걸이 | 6 | 없음, 스터드, 링형 등 |
| 악세사리 | 모자 | 8 | 없음, 비니, 왕관 등 |
| 악세사리 | 마스크 | 6 | 없음, 가면, 반다나 등 |
| 의상 | 상의 | 12 | 티셔츠, 후드, 갑옷, 망토 등 |
| 이펙트 | 배경 이펙트 | 8 | 없음, 불꽃, 벚꽃, 매트릭스 등 |

**렌더링 레이어 순서:** 피부 → 얼굴 피쳐(눈, 눈썹, 코, 입) → 헤어(뒤) → 의상 → 헤어(앞) → 악세사리(안경, 귀걸이, 마스크) → 모자

**주요 동작:**
- 파츠 선택 시 카드에 즉시 반영 (실시간 프리뷰)
- "랜덤 조합" 버튼으로 전체 파츠를 무작위 조합
- URL 직렬화: 14개 카테고리 값을 Base36 1문자씩 고정 순서로 14문자 직렬화 (`av=` 파라미터)
- [ ] 기능 3: **내보내기 및 바이럴 루프** — 상태 압축 URL 생성, 2D 썸네일 이미지(PNG) 다운로드, 카카오톡 공유 가이드, 뷰어 전용 모드 + CTA 버튼
- [ ] 기능 4: **마이크로 인터랙션** — 사운드 디자인(재질/뒤집기 효과음), 햅틱 피드백(모바일 진동), 초기 로딩 시퀀스 연출

## 4. 사용자 스토리

- 사용자로서 직관적인 에디터를 통해 나만의 3D 명함을 만들고 싶다. 그래서 카카오톡 오픈채팅방에서 강렬한 자기소개를 할 수 있다.
- 사용자로서 재질, 이펙트, 스탯 등을 변경할 때 실시간으로 카드에 반영되는 것을 보고 싶다. 그래서 즐거운 제작 경험을 느낄 수 있다.
- 사용자로서 완성된 카드를 URL 하나로 공유하고 싶다. 그래서 별도의 회원가입 없이 누구나 내 카드를 볼 수 있다.
- 공유받은 사용자로서 카드를 이리저리 돌려보며 감상하고 싶다. 그래서 "나도 만들어보고 싶다"는 욕구가 생긴다.
- 사용자로서 카드의 멋진 순간을 2D 이미지로 저장하고 싶다. 그래서 카카오톡에 이미지와 링크를 함께 올려 클릭률을 높일 수 있다.

## 5. 기술 스택

- **프론트엔드**: Vite + React + TypeScript
- **3D 렌더링**: React Three Fiber (Three.js)
- **상태 관리**: Zustand
- **CSS**: Tailwind CSS
- **백엔드(BaaS)**: Supabase (PostgreSQL, Auth, Storage, Edge Functions)
- **린트/포매팅**: Biome
- **테스트**: Vitest
- **배포**: Vercel (프론트엔드), Supabase (백엔드)

## 6. 비기능 요구사항

### 보안 요구사항
- 사용자 업로드 이미지 검증 및 크기 제한
- URL 파라미터 인젝션 방지 (상태 압축 URL 생성 시)
- Supabase RLS 정책 적용 (추론: 이미지 업로드 스토리지 등에 적용)

### 접근성 요구사항
- 키보드로 에디터 조작 가능
- 화면 리더 대응 (카드 내용 텍스트 대체)
- 모바일/데스크탑 반응형 지원

### 성능 요구사항
- 초기 로딩 3초 이내 (3D 에셋 포함)
- 3D 카드 인터랙션 60fps 유지
- 상태 압축 URL 길이 최소화 (카카오톡 URL 제한 고려)
- 번들 사이즈 최적화 (Three.js 트리 셰이킹)
